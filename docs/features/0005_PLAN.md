# Feature 0005: Compute progression metrics (e.g., crack growth rate, vegetation encroachment delta)

## Context

This feature implements the product-brief requirement, verbatim: **“Compute progression metrics (e.g., crack growth rate, vegetation encroachment delta).”**  
The backend already persists **detections** (with bounding boxes and optional centroids), **alignment pairs** linking baseline and target inspections per `asset_zone_id`, and **change events** for coarse state transitions. This feature adds a dedicated progression layer: derive quantitative metrics from aligned detection pairs over time, persist them for audit/query, and expose them via read APIs.

## Algorithms and behavior (step-by-step)

**A. Eligibility and inputs**

1. Require target inspection in `alignment_ready` (or equivalent) and at least one `Alignment` row with `change_type == persisted` and both `baseline_detection_id` and `target_detection_id` set.
2. Load baseline and target `Detection` rows and parent `Inspection` rows for timestamps.
3. Compute `delta_t` in seconds from chosen inspection timestamps; if below `progression_min_time_delta_seconds`, skip rate metrics or store only deltas without per-day normalization (choose one deterministic policy).

**B. Crack growth rate (example)**

1. For aligned pairs where `detection_type == defect` and `class_name` matches crack-like taxonomy (reuse or extend `class_taxonomy.py` with a `CRACK_CLASSES` set).
2. Compute size proxy \(s_b\), \(s_t\) from normalized bbox (same definition for both).
3. `crack_growth_rate = (s_t - s_b) / delta_t_days` (or per second then convert); store sign (negative = shrink proxy).
4. Persist `ProgressionMetric` with `metric_name=crack_growth_rate` and `payload` containing raw bbox components and `delta_t`.

**C. Vegetation encroachment delta (example)**

1. For aligned pairs where `detection_type == environmental_hazard` and class matches vegetation taxonomy.
2. Compute `area_b = (xmax-xmin)*(ymax-ymin)` in normalized space; same for target.
3. `vegetation_encroachment_delta = area_t - area_b`; optionally divide by `delta_t` for a rate variant stored as a second metric row (`vegetation_encroachment_rate`).

**D. Idempotency and reruns**

1. Before insert, `delete` existing `ProgressionMetric` rows for (`target_inspection_id`) to mirror alignment’s replace policy, or use upsert keyed by (`alignment_pair_id`, `metric_name`).
2. Recompute when alignment reruns for the same target inspection.

**E. API read behavior**

1. List endpoint returns metrics ordered by `asset_zone_id`, `metric_name`, `created_at`.
2. Support pagination and filter by `metric_name` and `asset_zone_id`.

## Phases
